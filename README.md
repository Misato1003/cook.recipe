# Cook.Recipe


(1) 目的　　

  　このアプリケーションは、料理の検索、登録、レビューを投稿することができます。
   
  　料理を作る際、何を作るのか悩んでいる時に、料理を検索することで、悩みを解決することができます。
   

(2) なぜ作ったのか

  　 仕事柄、献立作成をする際、この検索ボタンを使うことによって、献立作成に役立てると思ったからです。　　
   
   また、普段でも、料理を作る際、余っている食材で何を作ろうかな、今日は時間がないから料理時間が短く簡単にできる料理にしようかななど、そういう場合は、検索することで、より多くの悩みが解決できれば良いと思い作成しました。
   
   また、レビューをつけることによって、、レビューを見て、この料理を作ってみたいとか、作った感想をのせることで、より良い料理の情報が、広がれば良いと思い作成しました。
   
   また、料理を登録することで、より多くの人に、見てもらって、料理の情報を共有できたら良いと思い作成しました。そして、料理を共有することで、料理の輪が広がれば良いと思い作成しました。

(6) 実装した機能

* ユーザーの新規登録

* ユーザーのログイン、ログアウト

* ユーザの更新、編集

* パスワードを忘れた際の再設定

* レビューの登録、更新、編集、詳細、削除

* 料理のお気に入り登録

* 料理のお気に入り済みの表示

* 料理の検索


(7) 使い方

* お試し用に、テストログイン（閲覧用）を作りました。

　　テストログインでログインすることで、料理の登録、レビューの登録などができます。ただし、プロフィールの編集はできません。
     
* ログイン、新規登録

　　ログイン、新規登録は、メールアドレスと、パスワード（６文字以上）の設定が必要です。
   
* ログインした後

   ログインした後、ログイン画像が表示されるので、ログイン画像を触れることによって、ナビが表示されます。
   ナビには、プロフィールの設定、料理の新規登録、お気に入り済みの一覧表、ログアウトがあります。
  
* プロフィールの設定

　　プロフィールの設定は、ユーザーの名前、プロフィール画像、現在のパスワード、再設定のパスワード、確認用のパスワード、メールアドレス、一言紹介があります。
 　必須項目は、現在のパスワードです。
 　プロフィールの画像は、jpg,jpeg,gif,pngを登録することができます。
     
* ロゴをクリックすることで、トップページに行くことができます。

* おすすめ料理の５つの画像は、クリックすることで、検索結果の一覧が表示されます。

* 料理の検索は誰でも見ることができます。

   検索は、主に、料理名、食材、料理の分類、料理にかかった時間などが検索できます。また、それぞれを組み合わせて、検索することもできます。
   検索結果から、料理の詳細を見ることもできます。また、検索結果には、ID、料理名、料理にかかった時間の項目部分をクリックすることで、昇順や降順にすることができます。
   
* 料理の一覧表、料理の詳細は、誰でも、見ることができます。

* 料理の登録、更新、編集、削除は、登録したユーザーのみ、使用できます。
   
* 料理の登録、編集
   
   料理を登録する際は、料理名、食材、作り方は、必須入力です。
   画像は、jpg,jpeg,gif,pngを登録することができます。
   
* レビューの一覧表、レビューの詳細は、誰でも見ることができます。

* レビューの登録、更新、編集、削除は、登録したユーザーのみ、使用できます。

* レビューの登録、編集
    
    レビューの登録は、料理の詳細ページのところにあります。
    レビューを登録する際は、投稿者の名前、レビュー内容は、必須入力です。
    
* お気に入り
    
    料理の一覧表のところにお気に入りボタンがあるので、ボタンをクリックすることで、お気に入りの登録ができます。
    お気に入り登録をしたら、お気に入り済みの一覧表に表示されます。


(8) 苦労したこと

* リレージョンが難しかった。（特に、お気に入り済みの登録、一覧表示のところが難しかった。）
  
  一対多なのか、多対多なのか、考えながら、作成をすれば良かったです。紙に一回必要なDBの情報を書き出して、どれをどう繋ぐのか（外部キー）を考えながら、作業をすれば良かったです。
  
* rspecの書き方が分からず苦労しました。

  サイトを調べながら、rspecを書きました。request,controller,model,featuresなど、数多くのテストがあり、どれを使用すれば良いのかが分からなかったです。また、rspecの書き方が分からず、戸惑ってしまいました。
  
(9)こだわったこと
  
* 料理、レビューを登録する際、ユーザーがログインしないと、登録できないようにした。
* 料理の編集、更新、レビューの編集、更新は、登録したユーザーのみ編集、更新できるようにした。
* お気に入り登録することで、お気に入り済みの一覧表を表示させることで、自分だけのお気に入りの料理が、簡単に見つけられます。
* 検索する項目を多く増やしたことで、より条件に合うものが簡単に見つかるようにした。

(10)失敗したこと（今後、修正した方が良いと思うところ

*  料理の一覧表のところを、テーブル（表）でなくて、画像の下に料理名、料理のポイントなど分かりやすく表示した方が、スマホのレスポンシブ対応にした際、より見やすくなると思った。
